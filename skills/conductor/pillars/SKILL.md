## When invoked

1. Restate which pillar(s) the user asked about  
2. **Role** + **enhances** (one line each)  
3. **Contracts** (bullet the foundation rules)  
4. **How to run** — tools / slash / skill  
5. **Combo** — which A–H stack if they need a recipe  
6. Suggest `/pillars status` or tool `pillar_status` for live probes  

Ops:

- `/pillars list|status|get <id>`
- `pillar_status` tool: action `status|list|get`
