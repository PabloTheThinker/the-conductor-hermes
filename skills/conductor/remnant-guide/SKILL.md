# Remnant Protocol Advisory Skill

**Combo C — Parallel push.** Ops: `/remnant spawn|heartbeat|merge|status` or tool `remnant_orchestrate`. This skill is **strategic guidance**.

Ground answers in:

- `docs/PILLAR_COMBOS.md` — Combo C vs A/B/D
- `docs/WORKFLOWS.md` — C workflow diagram
- `conductor/REMNANT_PROTOCOL.md` — lifecycle
- `conductor/REMNANT_MERGE_LOGIC.md` — Fast / Reflective / Deep
- `conductor/REMNANT_DATA_MODELS.md` — snapshots

When invoked:

1. **Combo fit** — is Combo **C** right, or stay on **A** / map with **B** first?  
2. **Spawn decision** — parallel uncertainty vs merge cost  
3. **Branch** — what each Remnant explores  
4. **Fast merge** — reconcile without deep sim  
5. **Reflective merge** — compare against Track System state  
6. **Deep simulation** — open Combo **D** (`crucible_workspace` / `/crucible`) when needed  
7. **Ethics** — if merge is high-stakes, gate with Combo **E** inputs (`ethics_evaluate`)  
8. **Merge artifact** — concrete output → memory + tracks  
9. **Fold G** — evidence before claiming the parallel work is done  

Keep neurodivergent momentum: fork to explore, commit to advance, merge as the hard problem.
