# Conductor Review

Reference specs (use `research_view` for detail):

- `docs/PILLAR_COMBOS.md` — combo fit and decision tree
- `docs/WORKFLOWS.md` — expected workflow steps
- `governance/GOVERNANCE_SAFETY_AUDIT.md` — audit / escalation
- `governance/MAX_EFFORT_DELIBERATION.md` — high-stakes mode
- `memory/MEMORY_ARCHITECTURE.md` — memory fabric
- `SOUL.md` — Judgment: done = proven

When invoked, review the current thread or described work:

1. **What was attempted** — factual summary  
2. **Combo used vs needed** — which A–H was implied; should it have been another?  
3. **What succeeded** — evidence-based wins (artifacts, paths, logs — not narration alone)  
4. **Gaps** — missing Combo **G** verification, goal drift, skipped ethics on high-stakes, thrash without **F**  
5. **Recommendations** — prioritized fixes (max 5), including combo switch if wrong stack  
6. **Verdict** — continue / pivot / pause with one-line rationale  

Be direct. Flag goal drift and unfinished verification surfaces. Shipping claims without Combo G fail review.
